Custom Implementations for Spring Data Repositories

목적

  • Spring data repository 와 함께 커스텀한 Concreate repository 클래스를 만들어 같이 사용할때 Best Practice 만들기
  • 최소한의 boilerplate를 통해, 불필요한 추가 개발을 방지한다.
  • 추가적으로.. Layered architecture 가 멀티모듈로 적용된 프로젝트 구조에서 위 목적을 달성하기 위한 방법 정리

결과물

시나리오

Single module 에 적용

  • Domain package 에 아래 class 혹은 interface 가 모두 포함

Multi module 에 적용

  • Domian 과 Infrastructure 모듈이 분리 되어 있음

정리

참고에 추가한 Spring data jpa 공식 문서에서 언급한 내용처럼, 선언한 class 와 interface 의 postfix 에 따라서 내부적으로 custom generate 되게끔 구현되어 있으므로 관련 Rule을 지켜 구현하도록 한다.

  • *Repository
  • *Custom
  • *RepositoryImpl

참고

  • https://docs.spring.io/spring-data/jpa/docs/2.1.3.RELEASE/reference/html/#repositories.custom-implementations
